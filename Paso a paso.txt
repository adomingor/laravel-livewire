Primeros pasos

                                    --------------------------------------------------------
                                        GITHUB COMO PUBLICAR A UN REPOSITORIO POR 1RA VEZ
                                    --------------------------------------------------------

1. Preparación en GitHub
    Crea el repositorio en GitHub.
    Copia la URL HTTPS del repositorio que te proporciona GitHub (https://github.com/adomingor/laravel-livewire.git).

2. Preparación en VS Code (Casa)
    Abre tu carpeta de proyecto en Visual Studio Code.Abre la terminal integrada (Terminal -> New Terminal o Ctrl + ñ o Ctrl + j).

3. Conectar local con remoto y subirEjecuta los siguientes comandos en la terminal de VS Code:
    Inicializar Git: Crea un repositorio local.
        git init

    Añadir archivos: Prepara todos los archivos para el commit.
        git add .

    Confirmar cambios (Commit): Guarda la instantánea de los archivos.
        git commit -m "Iniciando repositorio desde casa php artisan make:livewire createPost"

    Vincular repositorio remoto: Conecta tu proyecto local con el repositorio de GitHub (reemplaza la URL).
        git remote add origin https://github.com/adomingor/laravel-livewire.git

    Subir código (Push): Envía el código a GitHub.
        git push -u origin main

cuando creo un componente con laravel livewire a los archivos le pone un rayo adelante del nombre ⚡create-post.blade.php por ejemplo, como evito eso

Ese rayo (⚡) es un emoji decorativo de Laravel Livewire para identificar componentes en el editor y se puede quitar desactivándolo en la configuración. Abre config/livewire.php y cambia 'render_on_load' => true, a false, o busca la opción class_namespace para personalizar el prefijo si usas una versión que lo permita

    No tengo livewire.php, hay que publicarlo

Publicar configuración: Si no tienes el archivo de configuración de Livewire, publícalo con:bashphp artisan vendor:publish --tag=livewire:config
Editar config/livewire.php: Busca la opción que controla los "stubs" (plantillas) o el prefijo de componente. En versiones modernas de Livewire, el rayo se puede eliminar modificando cómo Livewire genera los archivos.Alternativa - Personalizar Stubs: Puedes publicar los stubs (plantillas de archivos) de Livewire y quitar el rayo manualmente de ellos:bashphp artisan livewire:stubs
Luego, edita los archivos en la carpeta stubs/ que se acaba de crear en tu proyecto, eliminando el emoji ⚡ de los nombres de archivo.El emoji ⚡ es opcional y no afecta el funcionamiento de tu aplicación, solo sirve como indicador visual.
